argumentHint: [file-or-dir]

对代码进行安全审计扫描。

扫描规则：
1. **输入验证**：未验证的用户输入、SQL 拼接、命令注入
2. **认证授权**：硬编码密码、不安全的会话管理、权限绕过
3. **数据保护**：敏感信息明文存储、不安全的加密算法
4. **网络通信**：HTTP 明文传输、SSL 验证禁用、CORS 配置
5. **依赖安全**：已知漏洞的依赖版本
6. **配置安全**：调试模式开启、暴露的错误信息

输出：
- 每个发现标明：OWASP Top 10 类别、风险等级、受影响代码位置
- 提供具体的修复代码示例
- 生成安全改进清单
