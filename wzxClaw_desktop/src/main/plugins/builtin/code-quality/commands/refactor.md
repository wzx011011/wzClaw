description: 分析代码复杂度并给出重构建议
argumentHint: [file-path]
---

分析代码复杂度并提供重构建议。

步骤：
1. 读取目标文件
2. 分析以下指标：
   - 函数长度（超过 50 行标记）
   - 圈复杂度（嵌套层级超过 3 标记）
   - 参数数量（超过 4 个标记）
   - 重复代码块
   - 依赖关系

输出：
- 复杂度热力图：标记最复杂的函数
- 具体重构建议：
  - 提取函数（长函数拆分）
  - 简化条件（策略模式/早返回）
  - 减少参数（参数对象/Builder）
  - 消除重复（提取公共逻辑）
- 重构优先级排序（按影响面和收益）
