评估项目技术债务。

分析维度：
1. **代码质量债务**
   - TODO/FIXME/HACK 注释统计
   - 过长的文件和函数
   - 圈复杂度过高的函数
   - 重复代码

2. **测试债务**
   - 测试覆盖率估算
   - 关键路径缺少测试的模块
   - 测试质量和有效性

3. **架构债务**
   - 循环依赖
   - 过度耦合的模块
   - 违反 SOLID 原则的设计

4. **文档债务**
   - 缺少文档的公共 API
   - 过时的文档
   - 缺少 README 的模块

输出：
- 债务评分（0-100，越高越好）
- 按优先级排序的改进项（Quick Win / Short Term / Long Term）
- 预估每项改进的工作量（小时）
