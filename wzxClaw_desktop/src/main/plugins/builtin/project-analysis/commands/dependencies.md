---

分析项目依赖健康状况。

步骤：
1. 查找包管理文件（package.json / Cargo.toml / requirements.txt / go.mod / pubspec.yaml）
2. 分析直接依赖和开发依赖
3. 检查以下问题：
   - 过期版本（与最新版本对比）
   - 冗余依赖（已声明但未使用）
   - 安全漏洞（已知 CVE）
   - 许可证兼容性
   - 依赖体积

输出：
- 依赖概览：总数、直接/间接、许可证分布
- 问题列表：按严重程度排序
- 升级建议：安全的升级路径
- 清理建议：可安全移除的依赖
