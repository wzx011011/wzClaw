## Breaking Changes

- **CR-01**: App will no longer work without user-configured API key. First-launch must set key via Settings UI.
- **CR-03**: Permission request payload now includes `requestId` field. Renderer `PermissionRequest` component should pass it back in the response. Existing renderer code that ignores unknown fields will work — the fallback path handles missing requestId gracefully.
- **HI-06**: Mobile WebSocket clients should prefer `Sec-WebSocket-Protocol: wzxclaw-{token}` header. URL query param still works as fallback.
