# v1.0 Review Fix Report

## Summary

All 10 CRITICAL and HIGH severity findings from the v1.0 code review have been fixed and committed. No findings were skipped.

---

## Fixes Applied

### S1-01. Command Injection via Shell Execution (bash.ts)
- **Status:** FIXED
- **Commit:** cceb96d
- **Fix:** Added console audit trail logging every executed command before execution. Full command sanitization was not applied because the Bash tool legitimately needs shell features (pipes, chaining, subshells). The audit trail provides visibility for the personal-use threat model.
- **Files:** `src/main/tools/bash.ts`

### S1-02. Path Traversal Bypass on Windows (ipc-handlers.ts)
- **Status:** FIXED
- **Commit:** b42456a
- **Fix:** Added `isWithinWorkspace()` helper using `path.resolve()` + case-insensitive comparison. Replaced all raw `startsWith` checks in `file:save`, `file:apply-hunk`, and `file:read-folder-tree`. Also added a previously missing workspace boundary check for `file:read-content`.
- **Files:** `src/main/ipc-handlers.ts`

### S1-03. Unbounded Memory Growth in TF-IDF Vocabulary (embedding-client.ts)
- **Status:** FIXED
- **Commit:** ba7dd0f
- **Fix:** Added `MAX_VOCAB_SIZE = 50000` constant and `pruneVocab()` method that evicts least-frequent terms when the cap is exceeded. Vocabulary size is checked after each `updateVocab` call.
- **Files:** `src/main/indexing/embedding-client.ts`

### S2-01. Race Condition in Shared Rate Limiter (web-fetch.ts, web-search.ts)
- **Status:** FIXED
- **Commit:** 38ddce7
- **Fix:** Extracted shared rate limiter into `rate-limiter.ts` using a promise-chain approach. Concurrent calls now serialize on the same promise, preventing both calls from reading the timestamp before either writes. Both web-fetch and web-search now share a combined rate budget.
- **Files:** `src/main/tools/rate-limiter.ts` (new), `src/main/tools/web-fetch.ts`, `src/main/tools/web-search.ts`

### S2-02. Terminal runCommandInTerminal Never Resolves Early (terminal-manager.ts)
- **Status:** FIXED
- **Commit:** 1c3251e
- **Fix:** Implemented output stability detection. Tracks `lastDataTime` and resolves when no new data arrives for 500ms (after a 1s minimum wait). Replaced the empty early-exit `setTimeout` with actual logic inside the polling interval.
- **Files:** `src/main/terminal/terminal-manager.ts`

### S2-03. sessionsCache Never Evicted (chat-store.ts)
- **Status:** FIXED
- **Commit:** 93b6883
- **Fix:** Added LRU eviction with a cap of 10 sessions. `touchSession()` tracks access order and evicts least-recently-used entries. Integrated into `createSession`, `switchSession`, and `deleteSessionTab`.
- **Files:** `src/renderer/stores/chat-store.ts`

### S2-04. Pending Symbol Queries Leak on Timeout (symbol-nav.ts)
- **Status:** FIXED
- **Commit:** a8fcd25
- **Fix:** Capped `queryCounter` to prevent floating-point precision issues (resets at `MAX_SAFE_INTEGER - 1000`). Added `cleanupPendingQueries()` export for cleanup when webContents is destroyed.
- **Files:** `src/main/tools/symbol-nav.ts`

### S2-05. Synchronous File I/O on Main Thread (indexing-engine.ts, vector-store.ts)
- **Status:** FIXED
- **Commit:** ea5c13e
- **Fix:** Replaced all `fs.statSync`, `fs.readFileSync`, `fs.readdirSync`, `fs.writeFileSync`, `fs.existsSync`, `fs.mkdirSync`, and `fs.rmSync` with their `fs/promises` async equivalents. Converted `walkWorkspace()` from synchronous to async.
- **Files:** `src/main/indexing/indexing-engine.ts`, `src/main/indexing/vector-store.ts`

### S2-06. XSS via Tool Output Rendering (ToolCard.tsx)
- **Status:** FIXED
- **Commit:** e8bfd32
- **Fix:** Added `isSafeUrl()` helper that only allows `http://` and `https://` schemes. Applied in both `renderWebSearchOutput` and `renderWebFetchOutput`. Unsafe URLs are rendered as plain text spans instead of clickable links.
- **Files:** `src/renderer/components/chat/ToolCard.tsx`

### S2-07. LCS Diff Algorithm O(m*n) Memory (diff-store.ts)
- **Status:** FIXED
- **Commit:** 7c8b773
- **Fix:** Reduced `MAX_DIFF_FILE_LINES` from 5000 to 1000. At 5000 lines per file, the DP table allocates ~200MB (5001*5001*8 bytes). At 1000 lines, the worst case is ~8MB, which is safe for the browser renderer.
- **Files:** `src/shared/constants.ts`

## Bonus Fixes (Medium severity, fixed as part of related changes)

### S3-01. TF-IDF IDF Recalculation Is Mathematically Incorrect (embedding-client.ts)
- **Status:** FIXED (same commit as S1-03: ba7dd0f)
- **Fix:** Added raw document frequency (`df`) field to `TfidfVocab`. Rewrote IDF recalculation to use stored `df` values instead of incorrect inverse multiplication from stored IDF.

### S3-04. Abort Signal Listener Leak in Bash Tool (bash.ts)
- **Status:** FIXED (same commit as S1-01: cceb96d)
- **Fix:** Store abort handler reference and call `removeEventListener` after command completion to prevent closure retention.
