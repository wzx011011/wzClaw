# Phase 8 -- UI Design Contract

> Visual and interaction contract for Advanced Features (Terminal Panel + Web Tools + Symbol Navigation + Task/Plan System). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom CSS with CSS custom properties) |
| Icon library | none (text/Unicode symbols, matching existing pattern) |
| Font | -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif |

**Rationale:** The project uses a hand-rolled VS Code Dark+ theme with CSS custom properties defined in `ide.css`. No component library is used. All UI is plain React + CSS. This phase continues the same pattern. The `xterm.js` library provides its own theming API that maps to our color variables. The `cmdk` command palette already has a "Toggle Terminal" command registered as placeholder (D-88) -- this phase activates it.

## Detailed Component Specifications

### 1. Terminal Panel

**Trigger:** Ctrl+\` keyboard shortcut, command palette "Toggle Terminal", or menu bar View > Terminal.

**Location:** Bottom area of the IDE, below the editor panel. Uses nested Allotment for vertical resizing between editor area and terminal panel.

**Layout Structure:**
```
[Allotment vertical]
  [Editor Area (TabBar + EditorPanel/WelcomeScreen)]
  [TerminalPanel (resizable, default 200px, min 100px, max 60vh)]
[StatusBar (fixed bottom)]
```

**IDELayout Change:**
The middle Allotment.Pane (editor area) wraps in a nested vertical Allotment:
```tsx
<Allotment.Pane>
  <Allotment vertical defaultSizes={[70, 30]}>
    <Allotment.Pane minSize={200}>
      <div style={{ display: 'flex', flexDirection: 'column', height: '100%' }}>
        {hasTabs && <TabBar />}
        {hasTabs ? <EditorPanel /> : <WelcomeScreen />}
      </div>
    </Allotment.Pane>
    {showTerminal && (
      <Allotment.Pane minSize={100} maxSize={pane => Math.floor(pane.rect.height * 0.6)}>
        <TerminalPanel />
      </Allotment.Pane>
    )}
  </Allotment>
</Allotment.Pane>
```

**Terminal Panel Container:**
- Full width, fills allocated height from Allotment
- Display: flex, flex-direction: column
- Background: `var(--bg-secondary)` (#252526)
- Border-top: 1px solid `var(--border)` (#404040)

**Terminal Header/Tab Bar:**
- Height: 32px, min-height: 32px
- Background: `var(--bg-tertiary)` (#333333)
- Border-bottom: 1px solid `var(--border)`
- Layout: flex row, align-items: center
- Left: terminal tabs (TerminalTabs component)
- Right: action buttons

**Terminal Tab Bar Actions (right side):**
- "New Terminal" button: `+` symbol, same style as `.chat-tab-new` (transparent, 18px font, padding 4px 10px, hover: bg-secondary, color-primary)
- Split terminal button: not in scope (deferred)
- Maximize terminal button: not in scope (deferred)
- Kill terminal button: not shown -- use tab close button instead

### 3. xterm.js Terminal Instance

**Configuration:**
```typescript
const term = new Terminal({
  cursorBlink: true,
  cursorStyle: 'block',
  fontSize: 14,
  fontFamily: 'Consolas, "Courier New", monospace',
  theme: { /* see Color section above */ },
  allowProposedApi: true,
  scrollback: 1000,
  allowTransparency: false,
  lineHeight: 1.2,
})
```

**Addons:**
- `@xterm/addon-fit` -- auto-resize terminal to container dimensions on panel resize
- `@xterm/addon-web-links` -- make URLs in terminal output clickable (opens in system browser via shell.openExternal)

**Container:**
- Fills all available space below terminal tab bar
- Background: `#1e1e1e` (matches xterm theme background)
- No padding -- xterm renders edge-to-edge

**Resize Behavior:**
- On Allotment drag end: call `fitAddon.fit()` to resize terminal to new container size
- On terminal panel show/hide: call `fitAddon.fit()` after panel is mounted/visible
- Debounce resize calls at 100ms to prevent thrashing during drag

**PTY Connection:**
- When a terminal tab is created, spawn a new node-pty process (shell: bash on Linux/Mac, cmd.exe or powershell on Windows)
- PTY data is piped to terminal via `term.onData()` -> pty.write()
- PTY output is piped to terminal via pty.onData() -> term.write()
- Buffer last 65536 bytes (64KB) of PTY output for agent analysis (TERM-06)

**Agent Bash Tool Integration (TERM-05):**
- When Bash tool is called and terminal panel is visible and a terminal tab is active:
  - Write the command to the active PTY: `pty.write(command + '\r\n')`
  - Display a visual indicator in the terminal (thin colored line or marker) showing "Agent executing: {command}"
  - Tool result is populated from the buffered PTY output (capture output after command, detect prompt return)
- When terminal panel is not visible or no terminal is active:
  - Fall back to existing child_process execution (current Bash tool behavior)
  - No visual changes needed -- tool card shows result as before

### 5. Tool Cards for New Tools

All new tools (WebSearch, WebFetch, GoToDefinition, FindReferences, SearchSymbols) render using the existing ToolCard component pattern from `chat.css`. No new tool card components needed -- just new input/output rendering within the existing card.

**WebSearch ToolCard:**
- Header: "WebSearch" with search icon, tool path shows query
- Input section: search query string (monospace, 11px)
- Output section: list of search results, each showing:
  - Source URL (clickable link, color: `var(--accent)`)
  - Title text (13px, `var(--text-primary)`)
  - Snippet (11px, `var(--text-secondary)`, max 2 lines)
- Truncation: output truncated at 200px max-height with "Show more" button

**WebFetch ToolCard:**
- Header: "WebFetch" with link icon, tool path shows URL
- Input section: fetched URL (monospace, 11px, truncated with ellipsis)
- Output section: markdown-converted content (truncated at 15000 chars per CONTEXT.md)
  - Rendered as readable text in tool-card-section-content (same monospace style)
  - Link preserved at top: "Source: {url}" in accent color
- Truncation: output truncated at 200px max-height with "Show more" button

**Symbol Navigation ToolCards (GoToDefinition, FindReferences, SearchSymbols):**
- Header: operation name (e.g. "GoToDefinition"), tool path shows symbol name
- Input section: symbol name and optional context file path
- Output section: structured JSON result rendered as readable list:
  - Each result: file path (monospace, accent color), line number, symbol kind badge
  - Symbol kind badges: function (green), class (blue), variable (gray) -- small colored pills
- Truncation: if many results, show first 10 with "Show more" for remainder

## Interaction Contracts

### Terminal Panel

| Interaction | Behavior |
|-------------|----------|
| Ctrl+\` | Toggle terminal panel visibility (show/hide with Allotment animation) |
| Command palette "Toggle Terminal" | Same as Ctrl+\` |
| Click "New Terminal" button (+) | Create new terminal tab, spawn PTY, focus new tab |
| Click terminal tab | Switch to that terminal instance, focus it |
| Click tab close button | Kill PTY process, remove terminal tab. If last tab, close entire terminal panel |
| Drag Allotment sash (top of terminal) | Resize terminal panel height (100px min, 60vh max) |
| Agent runs Bash tool | If terminal visible: write command to active PTY, show agent indicator. If not visible: use child_process as before |
| Click URL in terminal | Open in system default browser (via xterm web-links-addon + shell.openExternal IPC) |
| Terminal panel hidden | All terminal instances are preserved (PTY processes keep running). No data loss on hide. |
| Terminal panel shown | All terminal instances are restored. Call fitAddon.fit() on each visible terminal. |

### Task Panel

| Interaction | Behavior |
|-------------|----------|
| Click "Tasks" button in chat header | Toggle task panel open/closed |
| Agent creates a task | Task panel auto-opens if closed, new task appears at bottom of list with animation |
| Agent updates task status | Status badge updates in real-time (transition animation on color change) |
| Task becomes blocked | Badge changes to "Blocked", tooltip shows blocker details |
| All tasks completed | Progress bar fills to 100%, "Tasks" button badge clears, progress text shows "All completed" |
| Click task item | No action (read-only for user) -- future: could jump to relevant code |
| Close task panel | Panel collapses, button remains in header with potential badge |

### WebSearch / WebFetch Tool Cards

| Interaction | Behavior |
|-------------|----------|
| Click source URL | Open URL in system browser (same as terminal URL handling) |
| Click "Show more" | Expand truncated output section (same as existing tool card show-more pattern) |
| Collapsed tool card | Shows tool name + status badge (same as existing ToolCard pattern) |

### Symbol Navigation Tool Cards

| Interaction | Behavior |
|-------------|----------|
| Click file path in result | Open that file in editor at the specified line (navigate to file + scroll to line) |
| Click "Show more" | Expand truncated results list |

## Accessibility Notes

- Terminal panel: xterm.js handles its own accessibility (screen reader mode via `aria-label` on container). Add `role="tabpanel"` and `aria-label="Terminal"` to the terminal container div.
- Task panel: task items should have `role="listitem"` within a `role="list"` container. Status badges should use `aria-label` (e.g. `aria-label="Status: In Progress"`).
- Focus management: when terminal panel opens, focus the active terminal. When terminal panel closes, return focus to previously focused element.
- Keyboard navigation in terminal: all keyboard input goes to PTY when terminal is focused (xterm.js handles this). Escape key does NOT close terminal (following VS Code behavior).
