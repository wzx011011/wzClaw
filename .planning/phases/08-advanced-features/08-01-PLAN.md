    2. Create `src/main/terminal/terminal-manager.ts`:
       - Export class `TerminalManager` (singleton pattern like WorkspaceManager)
       - Properties:
         - `private terminals: Map<string, { pty: IPty; buffer: string; cols: number; rows: number }>`
         - `private nextId: number = 1`
       - Constructor: initialize empty terminals map
       - Methods:
         - `createTerminal(cwd: string, cols: number = 80, rows: number = 24): string` -- spawns node-pty process with platform-appropriate shell (process.env.COMSPEC || 'cmd.exe' on Windows, process.env.SHELL || '/bin/bash' on others), stores in map with generated id `term-${this.nextId++}`, returns the id. On Windows, use `shell: true` in pty.spawn options. Set `name: 'xterm-256color'` for color support.
         - `killTerminal(id: string): void` -- kills PTY process, removes from map
         - `writeToTerminal(id: string, data: string): void` -- writes data to PTY stdin
         - `onTerminalData(id: string, callback: (data: string) => void): () => void` -- subscribes to PTY output, returns unsubscribe function. Appends data to buffer (keep max 65536 bytes = 64KB per TERM-06), calls callback.
         - `getOutputBuffer(id: string): string` -- returns current buffer content
         - `resizeTerminal(id: string, cols: number, rows: number): void` -- calls pty.resize()
         - `getActiveTerminalId(): string | null` -- returns first terminal id (or null)
         - `runCommandInTerminal(id: string, command: string): Promise<string>` -- writes command + '\r\n' to PTY, captures output for agent analysis. Returns a promise that resolves with the captured output after detecting the shell prompt returns. Use a simple timeout-based approach: capture output for up to 30 seconds, then return what was captured.
         - `dispose(): void` -- kills all terminals
       - Import `IPty` from `node-pty` and use `require('node-pty')` with try/catch for the import (native module may fail to load -- in that case, log warning and throw on createTerminal).
       - Buffer management: when new data arrives, append to buffer. If buffer exceeds 65536 bytes, trim from start to keep last 65536 bytes.
