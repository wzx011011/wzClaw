# Phase 6 -- UI Design Contract

> Visual and interaction contract for Foundation Upgrades (Session Persistence + Context Management + Command Palette). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom CSS with CSS custom properties) |
| Icon library | none (text/Unicode symbols, matching existing pattern) |
| Font | -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif |

**Rationale:** The project uses a hand-rolled VS Code Dark+ theme with CSS custom properties defined in `ide.css`. No component library is used. All UI is plain React + CSS. This phase continues the same pattern -- no new UI framework introduced. The `cmdk` library is unstyled/composable, meaning we provide all CSS ourselves, consistent with the existing approach.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, toggle spacing |
| sm | 8px | Compact element spacing, header padding vertical |
| md | 12px | Default element spacing, input padding, list item padding |
| lg | 16px | Section padding, modal body padding |
| xl | 24px | Layout gaps, section breaks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none

**Source:** Extracted from existing `ide.css` and `chat.css` patterns: headers use `padding: 8px 12px`, chat messages use `padding: 12px`, input areas use `padding: 8px 12px`. All existing values are multiples of 4.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 (normal) | 1.5 |
| Label | 11px | 600 (semibold) | 1.4 |

**Source:** Existing codebase standard: root font-size is 13px (ide.css line 35), headers use 11px semibold uppercase (ide.css lines 99-104), settings modal headings use 14px semibold (ide.css line 573), code/paths use 12px Consolas (ide.css line 163). Only 2 weights used project-wide: 400 and 600.

## Color

All colors inherit from the existing CSS custom property system defined in `ide.css`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg-primary)` = `#1e1e1e` | Editor background, code blocks, overlay backdrop |
| Secondary (30%) | `var(--bg-secondary)` = `#252526` | Chat panel background, sidebar, session list, command palette background |
| Accent (10%) | `var(--accent)` = `#007acc` | Send button, status bar, focus borders, active items, token bar fill, command palette selected item |
| Tertiary | `var(--bg-tertiary)` = `#333333` | Tool card headers, code block headers, command palette group headings |
| Destructive | `#f48771` | Error states, delete actions, over-limit token indicator |
| Warning | `#dcb67a` | Permission requests, near-limit token indicator (60-80% range) |
| Success | `#89d185` | Completed status, approve buttons, compact success indicator |

**Accent reserved for:** Send button, status bar, focus rings, selected command palette item, token bar fill under 60%, active session indicator.

**Token bar color thresholds:**
- Under 60%: `var(--accent)` = `#007acc` (blue, healthy)
- 60-80%: `#dcb67a` (amber, warning)
- Over 80%: `#f48771` (red, danger zone / approaching compact trigger)

**Source:** All colors extracted from existing `ide.css` custom properties and `chat.css` status badge colors. The warning/success/destructive colors are already established in tool-status and permission-request patterns.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (send) | Button shows play symbol, no text label (existing pattern) |
| Empty state (no sessions) | "No previous sessions" / subtitle: "Start a conversation to create one" |
| Empty state (no commands match) | "No matching commands" |
| Error state (session load failure) | "Failed to load session. The file may be corrupted." |
| Error state (compact failure) | "Compaction failed. Your conversation is unchanged." |
| Destructive action (delete session) | Confirmation: "Delete this session? This cannot be undone." -- two buttons: "Delete" (destructive style) and "Cancel" |
| Token usage display | "{current}K / {max}K tokens" -- e.g. "12.4K / 128K tokens" |
| Compact result | Inline message in chat: "Context compacted: 45.2K -> 8.1K tokens" |
| Command palette placeholder | "Type a command..." |
| Command palette trigger hint | Not shown in UI -- triggered by Ctrl+Shift+P keyboard shortcut |
| Session title fallback | First 50 characters of first user message, truncated with "..." |

**Source:** Empty state follows existing pattern in chat.css (`.chat-empty`). Confirmation dialogs follow existing permission-request pattern with approve/deny button pair. Token display format follows industry standard (Cursor, Claude Code).

## Detailed Component Specifications

### 1. Command Palette Overlay

**Trigger:** Ctrl+Shift+P (registered in IDELayout global keydown handler)

**Layout:**
- Fixed overlay centered horizontally, positioned at 20% from top of viewport
- Width: 520px
- Max height: 400px
- Background: `var(--bg-secondary)` (#252526)
- Border: 1px solid `var(--border)` (#404040)
- Border-radius: 6px
- Box-shadow: `0 8px 30px rgba(0, 0, 0, 0.5)`
- Backdrop behind: `rgba(0, 0, 0, 0.5)` (same as settings overlay)

**Search Input:**
- Full width, no border-radius (flat top corners matching dialog)
- Background: `var(--bg-primary)`
- Padding: `12px 16px`
- Font-size: 14px, color: `var(--text-primary)`
- Placeholder: "Type a command..." in `var(--text-secondary)`
- Focus: border-bottom 1px `var(--accent)`

**Command List:**
- Scrollable (overflow-y: auto, max-height ~340px)
- Grouped by category with heading labels
- Group heading: uppercase, 11px, semibold, `var(--text-secondary)`, padding `8px 16px 4px`
- Each item: full-width row, padding `6px 16px`, flex layout
  - Left: command label (13px, `var(--text-primary)`)
  - Right: keyboard shortcut badge (if defined)
- Hovered/selected item: background `var(--accent)` (#007acc), text white
- Empty state: centered text "No matching commands" in `var(--text-secondary)`, 13px

**Keyboard Shortcut Badge:**
- Display: `<kbd>` element
- Background: `var(--bg-tertiary)`
- Border: 1px solid `var(--border)`
- Border-radius: 3px
- Padding: `1px 6px`
- Font-size: 11px
- Color: `var(--text-secondary)`
- On selected row: border-color `rgba(255,255,255,0.3)`, color `rgba(255,255,255,0.8)`

**Built-in Commands (Phase 6):**

| Category | Command | Shortcut |
|----------|---------|----------|
| File | Open Folder | Ctrl+Shift+O |
| Session | New Session | -- |
| Session | Clear Session | -- |
| View | Toggle Sidebar | Ctrl+B |
| View | Toggle Terminal | Ctrl+` |
| Settings | Change Model | -- |
| Settings | Open Settings | -- |
| File | Save File | Ctrl+S |

**Animation:** Fade in overlay (opacity 0->1, 100ms ease). Fade out on close (100ms ease).

### 2. Session List

**Location:** Inside ChatPanel, between chat header and messages area. Collapsible panel toggled by a "History" button in the chat header.

**Collapsed state:** Not rendered (no space taken).

**Expanded state:**
- Max height: 200px, overflow-y: auto
- Background: `var(--bg-primary)`
- Border-bottom: 1px solid `var(--border)`
- Padding: 0

**Session Item:**
- Display: flex row, padding `8px 12px`, border-bottom 1px `var(--border)` (last item: no border)
- Left column (flex: 1):
  - Title line: 13px, `var(--text-primary)`, max 1 line, ellipsis overflow, max-width 200px
  - Timestamp line: 10px, `var(--text-secondary)`, relative time (e.g. "2 hours ago", "yesterday", "Apr 3")
- Right column:
  - Delete button: appears on hover, 18x18px, background transparent, text "x", color `var(--text-secondary)`, hover: `#f48771`
- Hover: background `var(--hover-bg)` (#2a2d2e)
- Active (currently loaded session): left border 2px `var(--accent)`, background `var(--active-bg)`

**Toggle Button:**
- Same style as existing `.chat-ctrl-btn` in chat header
- Label: "History" (text button)
- Badge: shows count of saved sessions if > 0 (e.g. "History (3)")

**Empty state:**
- Centered text "No previous sessions"
- Subtitle "Start a conversation to create one"
- Padding: 20px, color: `var(--text-secondary)`, font-size: 12px

**Delete Confirmation:**
- Inline confirmation replacing the session item content
- Text: "Delete this session? This cannot be undone."
- Two buttons: "Delete" (`.permission-btn-deny` style, red) and "Cancel" (`.chat-ctrl-btn` style)
- Auto-dismiss after 5 seconds if no action

### 3. Token Usage Indicator

**Location:** Inside chat header, between model selector and header controls. Inline element, does not wrap.

**Layout:**
- Display: inline-flex, align-items: center, gap: 6px
- Height: 20px (matches header row)

**Token Bar:**
- Width: 60px, height: 4px
- Background: `var(--bg-tertiary)` (#333333)
- Border-radius: 2px
- Fill: dynamic width based on percentage (0-100%)
  - Under 60%: `var(--accent)` (#007acc)
  - 60-80%: `#dcb67a` (amber)
  - Over 80%: `#f48771` (red)
- Transition: width 300ms ease

**Token Count Text:**
- Font-size: 10px
- Color: `var(--text-secondary)` normally
- Color changes to match bar color when over 60%
- Format: "{current}K / {max}K"
- Example: "12.4K / 128K"

**Tooltip (on hover):**
- Shows exact token count and percentage
- "12,432 / 128,000 tokens (9.7%)"
- Uses native `title` attribute for simplicity (no tooltip library)

**States:**
- No session / no messages: Bar hidden, text shows "-- / {max}K"
- Streaming: Bar updates in real-time as tokens accumulate
- After compact: Bar shrinks, text updates to new count

### 4. Compact Result Message

**Location:** Appears in the chat message list as a system-level message (not user or assistant).

**Visual Style:**
- Full-width (not a bubble), padding `8px 12px`
- Background: `rgba(137, 209, 133, 0.08)` (subtle green tint)
- Border-left: 3px solid `#89d185`
- Border-radius: 4px
- Font-size: 12px, color: `var(--text-secondary)`
- Content: "Context compacted: 45.2K -> 8.1K tokens"
- Margin: 8px 0

**Auto-compact (system-initiated):**
- Same style but border-left is `var(--accent)` (#007acc)
- Content: "Auto-compacted context: 45.2K -> 8.1K tokens (80% threshold reached)"

## Interaction Contracts

### Command Palette

| Interaction | Behavior |
|-------------|----------|
| Ctrl+Shift+P | Opens palette, focuses search input, lists all commands |
| Type in search | Fuzzy filters commands in real-time (cmdk built-in filter) |
| Arrow Up/Down | Navigate command list (cmdk built-in) |
| Enter | Execute selected command, close palette |
| Escape | Close palette without executing |
| Click outside palette | Close palette without executing |
| Click a command | Execute command, close palette |

### Session List

| Interaction | Behavior |
|-------------|----------|
| Click "History" button | Toggle session list panel open/closed |
| Click a session item | Load that session's messages, replace current chat content |
| Hover session item | Show delete button |
| Click delete button | Show inline confirmation |
| Click "Delete" in confirmation | Delete session file, remove from list, switch to new empty session if deleted session was active |
| Click "Cancel" in confirmation | Dismiss confirmation, return to normal item display |

### Token Indicator

| Interaction | Behavior |
|-------------|----------|
| Hover token bar/text | Show tooltip with exact count and percentage |
| Bar fills past 60% | Color transitions from blue to amber |
| Bar fills past 80% | Color transitions from amber to red |
| After compact | Bar shrinks to new token level, color returns to blue if under 60% |

### /compact Command

| Interaction | Behavior |
|-------------|----------|
| User types "/compact" and sends | Triggers manual compaction via IPC, input is not sent as normal message |
| Compaction succeeds | Compact result message appears in chat, token indicator updates |
| Compaction fails | Error banner appears: "Compaction failed. Your conversation is unchanged." |
