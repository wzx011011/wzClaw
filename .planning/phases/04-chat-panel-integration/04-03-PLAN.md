**settings-manager.ts** — New file. Per D-66, use Electron safeStorage for API key encryption.

```typescript
import { app, safeStorage } from 'electron'
import fs from 'fs'
import path from 'path'

interface StoredSettings {
  provider: string
  model: string
  baseURL?: string
  systemPrompt?: string
  // API keys stored separately, encrypted
}

export class SettingsManager {
  private settingsPath: string
  private keysPath: string
  private settings: StoredSettings
  private encryptedKeys: Map<string, Buffer> = new Map()

  constructor() {
    const userDataPath = app.getPath('userData')
    this.settingsPath = path.join(userDataPath, 'settings.json')
    this.keysPath = path.join(userDataPath, 'keys.enc')
    this.settings = { provider: 'openai', model: 'gpt-4o' }
  }

  load(): void
    - Read settings.json, parse into this.settings (use defaults if file missing)
    - Read keys.enc, parse as JSON of { provider: base64EncryptedBuffer }, decrypt each key
    - If safeStorage is not available (some Linux), store keys as plaintext (log warning)

  save(): void
    - Write this.settings to settings.json
    - Encrypt all API keys via safeStorage.encryptString, write to keys.enc as JSON

  getSettings(): { provider, model, hasApiKey, baseURL?, systemPrompt? }
    - Return current settings + hasApiKey boolean (check if encryptedKeys has current provider)

  updateSettings(request): void
    - Update fields from request
    - If request.apiKey: encrypt and store in encryptedKeys map
    - Call save()

  getApiKey(provider: string): string | undefined
    - Return decrypted key for provider

  getCurrentConfig(): { provider, model, apiKey, baseURL?, systemPrompt? }
    - Convenience method returning everything needed for LLM gateway configuration
}
```

**ipc-handlers.ts** — Replace the in-memory settings with SettingsManager:

1. Import and instantiate SettingsManager at module level
2. Call `settingsManager.load()` before registering handlers
3. Update `settings:get` handler: `return settingsManager.getSettings()`
4. Update `settings:update` handler: `settingsManager.updateSettings(request)`
5. The existing `apiKeys` Map and `currentSettings` object are replaced by SettingsManager
6. Update the `agent:send_message` handler: get API key from `settingsManager.getCurrentConfig()` instead of the old `currentSettings`
7. Important: The gateway needs the API key to make requests. Currently, the gateway is configured once at startup in main/index.ts. After this change, the agent:send_message handler should check if the gateway has the right provider configured and update it if settings changed. For MVP, a simple approach: before each agent run, call `gateway.addProvider(settingsManager.getCurrentConfig())` to ensure the gateway is up-to-date.

Also update the import in main/index.ts (or wherever the gateway is initialized) to use SettingsManager for initial configuration.

Note: The `hasApiKey` field in the getSettings response should be per-provider, not global. Change the response shape to include `hasApiKey: boolean` based on whether the current provider has a key stored.
