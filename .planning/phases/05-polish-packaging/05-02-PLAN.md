<task type="auto">
  <name>Task 1: Build NSIS installer and verify output</name>
  <files>dist/</files>
  <action>
1. Run the full build: `npm run build:win`. This compiles with electron-vite and then creates the NSIS installer.

2. Verify the output exists:
```bash
ls dist/wzxClaw*.exe
```
There should be a file like `wzxClaw Setup 0.1.0.exe`.

3. Check the installer size is reasonable (Electron apps are typically 80-200MB as installers).

4. Run the unpacked exe briefly to verify it launches without errors:
```bash
timeout 10 ./dist/win-unpacked/wzxClaw.exe 2>&1 || true
```
The app should open a window. It may fail to connect to LLM APIs (no keys configured in packaged build), but it should not crash on startup.

5. If the exe crashes on startup, check the logs:
   - On Windows, Electron logs go to stderr which the timeout command captures
   - Common fixes: missing asar unpack for native modules, incorrect main path, renderer index.html not found

6. If there are build or launch errors, fix electron-builder.yml or the relevant source and re-run. Document any fixes in the SUMMARY.
