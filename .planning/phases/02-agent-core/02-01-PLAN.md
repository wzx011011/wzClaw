    2. Create `src/main/tools/file-read.ts` implementing Tool:
       - name: "FileRead"
       - description: "Read the contents of a file. Returns file content with line numbers."
       - requiresApproval: false (read-only, per D-32)
       - inputSchema: `{ path: string (required), offset: number (optional, 0-based line offset), limit: number (optional, max lines to read) }`
       - Use Zod schema: `z.object({ path: z.string(), offset: z.number().optional(), limit: z.number().optional() })`
       - execute(): Use `fs.readFile` with utf-8. Split by newlines. Add line numbers with tab separator (format: `${lineNum}\t${line}`). Respect offset/limit parameters. Truncate at MAX_FILE_READ_LINES (from constants). Truncate final output string at MAX_TOOL_RESULT_CHARS. On file not found, return `{ output: "File not found: {path}", isError: true }`. Wrap in try/catch for other errors.

    3. Create `src/main/tools/grep.ts` implementing Tool:
       - name: "Grep"
       - description: "Search file contents by regex pattern. Returns matching lines with file path and line number."
       - requiresApproval: false (read-only, per D-32)
       - inputSchema: `{ pattern: string (required, regex), path: string (optional, directory to search, defaults to workingDirectory), include: string (optional, glob like "*.ts" to filter files) }`
       - Use Zod schema for validation
       - execute(): Use Node.js `child_process.execFileSync` to call `rg` (ripgrep) if available, otherwise fall back to a manual Node.js recursive file walk + regex match. Output format: `relative/path:lineNum:matchedLine\n`. Truncate at MAX_TOOL_RESULT_CHARS. On invalid regex, return isError=true with the regex error message. On no matches, return `{ output: "", isError: false }`.

    4. Create `src/main/tools/glob.ts` implementing Tool:
       - name: "Glob"
       - description: "Find files matching a glob pattern. Returns matching file paths."
       - requiresApproval: false (read-only, per D-32)
       - inputSchema: `{ pattern: string (required, glob like "**/*.ts"), path: string (optional, directory to search) }`
       - Use Zod schema for validation
       - execute(): Use Node.js `fs` to recursively walk directories matching the glob pattern. Convert the glob to a regex (handle `*` -> `[^/]*`, `**` -> `.*`, `?` -> `[^/]`). Return one path per line, relative to workingDirectory. Truncate at MAX_TOOL_RESULT_CHARS.

    5. Create `src/main/tools/tool-registry.ts`:

       - `getAll(): Tool[]` — returns all tools
       - `getDefinitions(): ToolDefinition[]` — maps tools to ToolDefinition[] (for passing to LLM Gateway StreamOptions.tools)
       - `getApprovalRequired(): string[]` — returns names of tools where requiresApproval === true
       - `createDefaultTools(workingDirectory: string): ToolRegistry` — factory that creates and registers all 6 tools (FileRead, FileWrite, FileEdit, Bash, Grep, Glob). For THIS task, only register FileRead, Grep, Glob. FileWrite, FileEdit, Bash will be added in Plan 02. Import their classes and register them.
