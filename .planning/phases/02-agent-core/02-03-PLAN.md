<objective>
Build the Agent Loop that drives multi-turn LLM conversations with tool execution, loop detection, cancellation, and context management.

Purpose: This is the brain of the application. The agent loop orchestrates the entire flow: send user message to LLM, parse tool calls from the response, execute tools (with permission checks for destructive ops), feed results back to LLM, and repeat until the LLM returns a final text response or hits a safety guard.
Output: AgentLoop class, LoopDetector, MessageBuilder, and all associated tests.
</objective>

  <action>
    1. Create `src/main/agent/agent-loop.ts`:
       - Import dependencies: LLMGateway from llm/gateway, ToolRegistry from tools/tool-registry, PermissionManager from permission/permission-manager, LoopDetector from ./loop-detector, MessageBuilder from ./message-builder, AgentEvent/AgentConfig from ./types, StreamOptions from llm/types, Message/ToolCall/StreamEvent from shared/types, MAX_AGENT_TURNS from shared/constants.
       - `AgentLoop` class:
         - Constructor takes: `gateway: LLMGateway`, `toolRegistry: ToolRegistry`, `permissionManager: PermissionManager`
         - `private messages: Message[]` — in-memory message array per D-24
         - `private loopDetector: LoopDetector`
         - `private messageBuilder: MessageBuilder`
         - `private abortController: AbortController | null`
         - `async *run(userMessage: string, config: AgentConfig, sender?: Electron.WebContents): AsyncGenerator<AgentEvent>`:
           a. Create AbortController, store reference. Pass abortSignal to all downstream calls.
           b. Add user message to internal messages array: `{ role: 'user', content: userMessage, timestamp: Date.now() }`
           c. Build system prompt using messageBuilder.buildSystemPrompt(config.systemPrompt, toolRegistry.getDefinitions())
           d. Detect provider from config.model (reuse gateway's detectProvider logic: claude* -> anthropic, else -> openai)
           e. Loop (max config.maxTurns ?? MAX_AGENT_TURNS iterations):
              - Build provider-specific messages using messageBuilder.buildMessages(this.messages, provider)
              - Build StreamOptions: `{ model, messages: providerMessages, systemPrompt, tools: toolRegistry.getDefinitions(), abortSignal: this.abortController.signal }`
              - Call `gateway.stream(streamOptions)` — iterate over StreamEvents:
                - text_delta: yield `{ type: 'agent:text', content }`
                - tool_use_start: track tool call accumulation
                - tool_use_end: record ToolCall, add to assistant message
                - error: yield `{ type: 'agent:error', error, recoverable: false }`, break loop
                - done: record usage, exit inner loop
              - After streaming one LLM response completes:
                - If no tool calls in response: yield `{ type: 'agent:done', usage, turnCount }`, return
                - If tool calls present: for each tool call:
                  - yield `{ type: 'agent:tool_call', toolCallId, toolName, input }`
                  - Check loopDetector.isLooping() -> if true: yield `{ type: 'agent:error', error: 'Loop detected: same tool call repeated 3+ times', recoverable: true }`, break outer loop
                  - loopDetector.record(name, input)
                  - Look up tool in registry. If not found: create ToolResult with error message
                  - If tool.requiresApproval: check permissionManager (with sender). If denied: create ToolResult with "Permission denied" message
                  - If tool found and approved: execute tool with context `{ workingDirectory: config.workingDirectory, abortSignal }`. Wrap in 30s timeout per D-36.
                  - Create ToolResultMessage, add to messages
                  - yield `{ type: 'agent:tool_result', toolCallId, toolName, output, isError }`
                - Continue outer loop (feed tool results back to LLM)
           f. If max turns exceeded: yield `{ type: 'agent:error', error: 'Max agent turns exceeded (${turnCount})', recoverable: true }`
           g. After loop: yield `{ type: 'agent:done', usage, turnCount }`

         - `cancel(): void` — calls `this.abortController?.abort()`, resets loopDetector

         - `reset(): void` — clears messages array, resets loopDetector

         - `getMessages(): Message[]` — returns copy of messages array (for IPC forwarding to renderer in Phase 4)

    2. Update `src/main/tools/tool-registry.ts`:

         c. "detects loop after 3 identical tool calls" — gateway yields same tool call 3 times -> agent yields agent:error with loop message
         d. "stops at max turns" — gateway always returns tool calls -> agent stops at MAX_AGENT_TURNS and yields agent:error
         e. "cancels mid-stream" — call cancel() while streaming -> generator stops
         f. "handles tool not found" — gateway returns tool call for unknown tool -> agent yields agent:tool_result with isError=true

         h. "handles permission denied" — permissionManager.requestApproval returns false -> agent yields agent:tool_result with "Permission denied" message
         i. "auto-approves read-only tools" — gateway returns FileRead tool call -> permissionManager.requestApproval is NOT called
