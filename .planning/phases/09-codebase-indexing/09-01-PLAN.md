    Export class `CodeChunker`:

    - Splits at boundaries: `function`, `class`, `interface`, `type`, `export default`, and `export const/function/class` declarations
    - Uses regex: `/^(export\s+)?(default\s+)?(async\s+)?function\s+/m`, `/^(export\s+)?(default\s+)?class\s+/m`, `/^(export\s+)?interface\s+/m`, `/^(export\s+)?type\s+\w+\s*=/m`, `/^(export\s+)?const\s+\w+\s*=/m`
    - Falls back to splitting at blank lines (groups of 2+ newlines) if no boundaries found
    - If a single chunk exceeds 512 estimated tokens (chars/4), splits it further at blank lines
    - Minimum chunk size: 20 characters (skip tiny fragments like `}`)
    - Each chunk stores relative filePath, startLine, endLine, content, language, tokenEstimate

    Export class `VectorStore`:
    - `constructor(workspaceRoot: string)` -- creates `.wzxclaw/index/` directory inside workspace root
    - `upsert(entries: IndexEntry[]): Promise<void>` -- merge entries into storage, write to `vectors.jsonl` (one JSON object per line, append-only with periodic compaction)
    - `deleteByFile(filePath: string): Promise<void>` -- remove all entries for a given file (rewrite vectors.jsonl excluding matches)
    - `search(queryEmbedding: number[], topK: number = 10): SearchResult[]` -- load all entries, compute cosine similarity, return top-K results sorted by score descending
    - `getEntryCount(): Promise<number>` -- count total indexed entries
    - `clear(): Promise<void>` -- delete entire .wzxclaw/index/ directory
    - `loadAll(): Promise<IndexEntry[]>` -- load all entries from vectors.jsonl

    Cosine similarity implementation (inline, no library):
    ```
    function cosineSimilarity(a: number[], b: number[]): number {
      let dot = 0, normA = 0, normB = 0
      for (let i = 0; i < a.length; i++) {
        dot += a[i] * b[i]
        normA += a[i] * a[i]
        normB += b[i] * b[i]
      }
      return dot / (Math.sqrt(normA) * Math.sqrt(normB))
    }
    ```

    Storage format (per CONTEXT.md D: "Simple JSON file-based storage in .wzxclaw/index/"):
    - `.wzxclaw/index/vectors.jsonl` -- one IndexEntry per line (newline-delimited JSON)
    - `.wzxclaw/index/meta.json` -- `{ version: 1, created: timestamp, fileCount: number }`
    - No database dependency, no sql.js, no sqlite-vec (CONTEXT.md decision overrides REQUIREMENTS.md IDX-03)

    **src/main/indexing/embedding-client.ts:**

    Export interfaces:
    ```typescript
    export interface EmbeddingResult {
      embedding: number[]
      type: 'api' | 'tfidf'
    }

    export interface EmbeddingClientConfig {
      apiKey?: string
      baseURL?: string        // embedding API base URL (e.g., https://api.openai.com/v1)
      model?: string          // embedding model name (e.g., text-embedding-3-small)
    }
    ```

    Export class `EmbeddingClient`:
    - `constructor(config: EmbeddingClientConfig)`
    - `isAvailable(): boolean` -- returns true if apiKey and baseURL are configured
    - `async embed(text: string): Promise<EmbeddingResult>`
      - If API available: call `POST {baseURL}/embeddings` with `{ input: text, model }` using native fetch (NOT the openai SDK, to avoid import overhead -- use fetch directly with JSON parsing)
      - Parse response: `response.data[0].embedding` (OpenAI format)
      - Embedding model default: `text-embedding-3-small` (1536 dimensions)
      - Batch embedding: `async embedBatch(texts: string[]): Promise<EmbeddingResult[]>` -- POST with `input: texts` array, much more efficient
      - If API unavailable or fails: fall back to TF-IDF
      - Rate limit: max 20 requests per second (100ms delay between batches)
    - TF-IDF fallback (local, no API needed):
      - Tokenize: lowercase, split on non-alphanumeric, remove stop words (common 100 English stop words)
      - Build vocabulary from all unique terms across chunks (maintained as `Map<string, number>`)
      - Compute TF-IDF vector: term frequency * inverse document frequency
      - Vocabulary must be persisted so search queries can be vectorized against the same vocabulary
      - Store vocabulary in `.wzxclaw/index/tfidf-vocab.json`
      - TF-IDF vectors are sparse (only non-zero terms) -- store as `{ term: number }` map, convert to dense array for cosine similarity at search time

    **src/main/indexing/indexing-engine.ts:**

    Export interfaces:
    ```typescript
    export type IndexingStatus = 'idle' | 'indexing' | 'ready' | 'error'

    export interface IndexingProgress {
      status: IndexingStatus
      fileCount: number       // total files indexed
      currentFile: string     // file currently being indexed (empty if idle/ready/error)
      error?: string
    }
    ```

    Export class `IndexingEngine`:
    - `constructor(workspaceRoot: string, embeddingClient: EmbeddingClient)`
    - `async indexFull(): Promise<void>` -- walk workspace directory, chunk all eligible files, embed, store
    - `async indexFile(filePath: string): Promise<void>` -- chunk and index a single file (for incremental updates)
    - `async removeFile(filePath: string): Promise<void>` -- remove entries for a deleted file
    - `async search(query: string, topK?: number): Promise<SearchResult[]>` -- embed query, search vector store
    - `getStatus(): IndexingProgress`
    - `onProgress(callback: (progress: IndexingProgress) => void): () => void` -- subscribe to status changes, returns unsubscribe
    - `dispose(): void` -- cleanup

    Indexing workflow:
    1. Set status to 'indexing'
    2. Walk workspace directory recursively (same skip dirs as VectorStore)
    3. For each file: check mtime against VectorStore, skip if unchanged (incremental per IDX-04)
    4. Read file content, detect language (reuse getLanguageFromPath from workspace-manager.ts)
    5. Skip if >100KB or binary extension
    6. Chunk file using CodeChunker
    7. Embed chunks using EmbeddingClient (batch embedding: collect up to 50 chunks, embed batch, then store)
    8. Upsert into VectorStore
    9. Update progress (fileCount, currentFile)
    10. On completion: set status to 'ready'
    11. On error: set status to 'error' with error message

    Search workflow:
    1. Embed the query string using EmbeddingClient
    2. Call VectorStore.search(queryEmbedding, topK)
    3. Return results with file paths relative to workspace root

    Performance considerations:
    - Batch embeddings: collect chunks into batches of 50, embed batch, then store
    - 100ms delay between API calls to respect rate limits
    - Skip files that haven't changed (compare mtime with stored mtime in IndexEntry)
    - If a file's mtime changed, delete old entries first, then re-index (deleteByFile + upsert)
