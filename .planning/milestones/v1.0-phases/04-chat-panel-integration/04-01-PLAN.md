**chat-store.ts** — Zustand store following the same pattern as tab-store.ts:

State interface:
- `messages: ChatMessage[]` where ChatMessage is a local type:
  ```typescript
  interface ChatMessage {
    id: string           // uuid
    role: 'user' | 'assistant' | 'tool_result'
    content: string      // for user and assistant; for tool_result: the output
    timestamp: number
    // assistant-only fields:
    toolCalls?: { id: string; name: string; status: 'running' | 'completed' | 'error'; input?: Record<string, unknown>; output?: string; isError?: boolean }[]
    isStreaming?: boolean  // true while this assistant message is receiving tokens
    usage?: { inputTokens: number; outputTokens: number }
  }
  ```
- `conversationId: string` — uuid, generated on first message or on clear
- `isStreaming: boolean` — true while agent is generating
- `error: string | null`

Actions:
- `init()` — subscribes to all 5 stream events (onStreamText, onStreamToolStart, onStreamToolResult, onStreamEnd, onStreamError). Returns unsubscribe function. Follow the same `useEffect` + cleanup pattern used in IDELayout.tsx for onFileChanged.
  - onStreamText: append `content` to the last assistant message's content (find last message with role='assistant' and isStreaming=true; if none exists, create one first)
  - onStreamToolStart: find last assistant message, add a toolCall entry with `{ id, name, status: 'running' }`
  - onStreamToolResult: find the toolCall by id in any assistant message, set its output, isError, and status to 'completed' or 'error'
  - onStreamEnd: set isStreaming=false on last assistant, store usage
  - onStreamError: set `error` state, set isStreaming=false
- `sendMessage(content: string)` — pushes a user ChatMessage, creates an empty assistant ChatMessage with isStreaming=true, calls `window.wzxclaw.sendMessage({ conversationId, content })`. Catches errors and sets error state.
- `stopGeneration()` — calls `window.wzxclaw.stopGeneration()`, sets isStreaming=false
- `clearConversation()` — resets messages to [], generates new conversationId, clears error

**settings-store.ts** — Zustand store:
- State: `provider: string`, `model: string`, `hasApiKey: boolean`, `baseURL?: string`, `systemPrompt?: string`, `isLoading: boolean`
- Actions:
  - `loadSettings()` — calls `window.wzxclaw.getSettings()`, populates state
  - `updateSettings(request)` — calls `window.wzxclaw.updateSettings(request)`, then calls loadSettings to refresh
  - `getModelLabel()` — looks up model name from DEFAULT_MODELS constant in shared/constants.ts

Import `v4 as uuidv4` from 'uuid' for ID generation (same pattern as tab-store.ts).
Import DEFAULT_MODELS from shared/constants.
