<objective>
Implement web search/fetch tools and code symbol navigation tools. WebSearch uses configurable search backends (LLM built-in or manual API key for Brave/SerpAPI). WebFetch fetches URLs and converts HTML to markdown. SymbolNav provides GoToDefinition, FindReferences, and SearchSymbols via Monaco's TypeScript worker accessed through IPC.

Purpose: Gives the agent the ability to search the web for information and navigate code symbols -- essential capabilities for an AI coding assistant.
Output: 3 new tools registered in ToolRegistry, IPC channel for symbol queries, ToolCard rendering for all new tool types.

         - Return `{ output: formattedContent, isError: false }`
         - On error (status code != 200, network error): return `{ output: 'Web fetch failed: {status} {error}. Response: {first 500 chars}', isError: true }`

         - execute: calls `senderFn()?.send('symbol:query', { queryId, operation: 'goto-definition', params: { symbolName, filePath } })`, then waits for response via a promise that resolves when the renderer sends back `symbol:result` with matching queryId.
