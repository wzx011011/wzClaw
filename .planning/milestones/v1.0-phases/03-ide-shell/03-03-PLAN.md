     - Gets the tab by id
     - Calls `await window.wzxclaw.saveFile({ filePath: tab.filePath, content: tab.content })`
     - On success: update `diskContent = content`, `isDirty = false`
     - On error: log error, keep dirty state (user can retry)
   - Add `openOrRefreshTab(filePath: string)` action:
     - If a tab with this filePath already exists: re-read file content from disk via `window.wzxclaw.readFile({ filePath })`, update both content and diskContent, keep the tab active
     - If no tab exists: read file content, call openTab to create new tab and make it active
     - This is the entry point for agent edit auto-refresh per D-52
   - Add `handleExternalFileChange(filePath: string, changeType: string)` action:
     - If changeType === 'modified' or 'created' and a tab with this filePath is open: re-read file content and update the tab (refreshTabContent)
     - If changeType === 'deleted' and a tab is open: optionally close the tab or mark it as deleted
     - If changeType === 'modified' and the tab is dirty (user has unsaved edits): do NOT overwrite -- instead show a notification or skip (avoid losing user work)

Per D-52 (agent edits trigger auto-open/refresh):

The agent executes FileWrite and FileEdit tools through the AgentLoop. When these tools succeed, we need to notify the renderer so it can refresh the corresponding editor tab.

1. **Update src/main/ipc-handlers.ts:**
   In the agent:send_message handler, after processing agent:tool_result events, check if the tool was FileWrite or FileEdit and if the result was successful (isError === false):

   ```typescript
   case 'agent:tool_result': {
     sender.send(IPC_CHANNELS['stream:tool_use_end'], {
       id: agentEvent.toolCallId,
       parsedInput: {
         output: agentEvent.output,
         isError: agentEvent.isError,
         toolName: agentEvent.toolName,
       },
     })

     // Forward file changes from agent tool execution to renderer
     if (!agentEvent.isError && (agentEvent.toolName === 'FileWrite' || agentEvent.toolName === 'FileEdit')) {
       // Re-read the file to get fresh content and send to renderer
       // We need the file path from the tool's input. The agent event doesn't
       // carry the original input here, but we can extract it from the tool call.
       // Alternative: just send a file:changed event so the renderer re-reads from disk.
       sender.send('file:changed', {
         filePath: agentEvent.filePath, // need to add this to AgentToolResultEvent
         changeType: 'modified'
       })
     }
     break
   }
   ```

   Problem: The current AgentToolResultEvent doesn't carry the file path. We have two options:
   - Option A: Add the input to AgentToolResultEvent so we can extract the path
   - Option B: Track tool calls by ID and look up the input from the earlier agent:tool_call event

   Choose Option B (track tool calls): In the IPC handler, maintain a `Map<string, Record<string, unknown>>` mapping toolCallId to input. On agent:tool_call, store the input. On agent:tool_result, look up the input to extract the path.

   ```typescript
   const toolCallInputs = new Map<string, Record<string, unknown>>()

   // In agent:tool_call case:
   toolCallInputs.set(agentEvent.toolCallId, agentEvent.input)

   // In agent:tool_result case:
   if (!agentEvent.isError && (agentEvent.toolName === 'FileWrite' || agentEvent.toolName === 'FileEdit')) {
     const toolInput = toolCallInputs.get(agentEvent.toolCallId)
     const filePath = toolInput?.path as string | undefined
     if (filePath) {
       const absolutePath = path.isAbsolute(filePath) ? filePath : path.resolve(config.workingDirectory, filePath)
       sender.send('file:changed', {
         filePath: absolutePath,
         changeType: 'modified'
       })
     }
   }
   ```
